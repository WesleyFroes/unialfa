Paradigmas de Programação

O que é Paradigmas de Programação

Um paradigma de programação e um padrão ou modelo de um software, e um conjunto de ideias conceitos de design e normas que definem a maneira como o código de programação e escrito e organizado. Os paradigmas de programação são frequentemente diferenciados pelas técnicas de programação que proíbem ou permitem. Por exemplo, a programação estruturada não permite o uso de goto. Esse e um dos motivos pelo qual novos paradigmas são considerados mais rígidos que estilos tradicionais

Principais paradigmas de programação

  • Imperativo: O Programa e escrito como uma sequencia de instruções que indicam ao computador o que fazer..
  • Declarativo: O programa e escrito como uma descrição do que o computador deve fazer, sem especificar como fazer.
  • Funcional: O paradigma e escrito como uma composição de funções que não tem efeitos colaterais.
  • Logico: O programa e escrito como um conjunto de provas que demonstram a validade do resultado.
  • Orientado a Objeto: O programa e escrito como um conjunto de objetos que interagem entre si.
  • Orientada a Evento: O programa e escrito como uma resposta a eventos externos.

    • Exemplos

    Imperativo: C, C++, Java, Python, JavaScript
  • Declarativo: SQL, R, Prolog
  • Funcional: Haskell, Lisp, Scala
  • Lógico: Prolog
  • Orientado a objetos: Java, Python, C++, C#
  • Orientado a eventos: JavaScript, HTML5, CSS
